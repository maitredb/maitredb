# v0.3.0 — DuckDB, ClickHouse, Arrow Result Model, Streaming Maturity

## Context

v0.2.0 delivered production-ready connection management (pooling, caching, history) on top of the three initial drivers (SQLite, PostgreSQL, MySQL). v0.3.0 adds two analytics-oriented databases and migrates the internal result model to Apache Arrow — the foundation for zero-copy, columnar-native data flow throughout the stack.

The architecture spec defines Arrow as the internal format from the start. v0.3.0 is where that becomes real: DuckDB returns Arrow natively, ClickHouse streams Arrow IPC over HTTP, and all existing drivers gain a conversion layer so consumers see a uniform `RecordBatch`-based API.

---

## Execution Order

**3c (Arrow model) → 3a (DuckDB) → 3b (ClickHouse) → 3d (Streaming/backpressure)**

Arrow types land first because drivers depend on them. DuckDB before ClickHouse (simpler: embedded, no server, easier to test). Streaming maturity wraps everything with the `--stream` CLI flag and batch-size control.

## Phase 1: Arrow Utilities and Executor Updates (`@maitredb/core`)

### 1.1 Create `packages/core/src/arrow-utils.ts`

```typescript
import {
  RecordBatch,
  Schema,
  Field,
  Utf8,
  Float64,
  Bool,
  Int64,
  tableFromArrays,
  type DataType,
} from 'apache-arrow';
import type { FieldInfo, MaitreType } from '@maitredb/plugin-api';

/**
 * Convert a flat array of JS objects + FieldInfo metadata into an Arrow RecordBatch.
 * Used by non-Arrow drivers after buffering a chunk of rows.
 * Type mapping: MaitreType → Arrow DataType (best-effort; falls back to Utf8).
 */
export function rowsToRecordBatch(
  rows: Record<string, unknown>[],
  fields: FieldInfo[]
): RecordBatch;

/**
 * Materialise an Arrow RecordBatch into an array of plain JS objects.
 * Reads each column vector sequentially — one allocation per row.
 */
export function recordBatchToRows(batch: RecordBatch): Record<string, unknown>[];

/**
 * Async generator: reads rows from a JS-object async iterable, buffers up to
 * batchSize rows, then yields a RecordBatch. Fields are inferred from the first
 * row if the FieldInfo array is empty.
 */
export async function* batchRows(
  source: AsyncIterable<Record<string, unknown>>,
  fields: FieldInfo[],
  batchSize: number
): AsyncIterable<RecordBatch>;

/**
 * Map a MaitreType to the closest Arrow DataType for schema inference.
 */
export function maitreTypeToArrow(type: MaitreType): DataType;
```

**Implementation notes for `rowsToRecordBatch`:**
- Build column arrays from `rows` using `fields` for type hints
- Use `tableFromArrays({ col1: [...], col2: [...] })` then grab the first (and only) batch
- For numeric columns: use `Float64Array`; for integer: `BigInt64Array`; for bool: `Uint8Array` (Arrow Bool); for everything else: `Array<string | null>`

**Implementation notes for `batchRows`:**
- Accumulate rows into `buf: Record<string, unknown>[]`
- When `buf.length >= batchSize`, call `rowsToRecordBatch(buf, fields)` and yield, then reset `buf`
- After iteration ends, yield remaining rows if `buf.length > 0`
- Infer `fields` from `Object.keys(firstRow)` if the provided array is empty

### 1.2 Create `packages/core/src/arrow-result.ts`

```typescript
import type { RecordBatch, Vector } from 'apache-arrow';
import type { QueryResult } from '@maitredb/plugin-api';
import { recordBatchToRows } from './arrow-utils.js';

const TOOBJECTS_WARN_THRESHOLD = 10_000;

/**
 * Ergonomic wrapper around a QueryResult that carries an Arrow RecordBatch.
 * Provides column-level access and a Proxy-based lazy row iterator.
 */
export class ArrowResult {
  constructor(private readonly _result: QueryResult) {
    if (!_result.batch) throw new Error('ArrowResult requires result.batch');
  }

  /** Access a named column as an Arrow Vector. O(1). */
  column(name: string): Vector {
    const col = this._result.batch!.getChild(name);
    if (!col) throw new Error(`Column '${name}' not found`);
    return col;
  }

  /**
   * Lazy Proxy-based row iterator. Each property access reads from the column
   * buffer — no per-row object allocation until values are read.
   */
  rows(): Iterable<Record<string, unknown>> {
    const batch = this._result.batch!;
    return {
      [Symbol.iterator]() {
        let idx = 0;
        return {
          next() {
            if (idx >= batch.numRows) return { done: true as const, value: undefined };
            const i = idx++;
            const row = new Proxy({ __i: i } as Record<string, unknown>, {
              get(target, prop: string) {
                if (prop === '__i') return target.__i;
                const col = batch.getChild(prop);
                return col?.get(i) ?? undefined;
              },
              ownKeys() {
                return batch.schema.fields.map(f => f.name);
              },
              has(_t, prop: string) {
                return batch.getChild(prop) !== null;
              },
            });
            return { done: false, value: row as Record<string, unknown> };
          },
        };
      },
    };
  }

  /**
   * Materialise all rows as plain JS objects.
   * Logs a warning to stderr if rowCount exceeds 10,000.
   */
  toObjects(): Record<string, unknown>[] {
    if (this._result.rowCount > TOOBJECTS_WARN_THRESHOLD) {
      process.stderr.write(
        `[maitredb] ArrowResult.toObjects() called on ${this._result.rowCount} rows — ` +
        `consider using .rows() for lazy iteration.\n`
      );
    }
    return recordBatchToRows(this._result.batch!);
  }

  get raw(): QueryResult { return this._result; }
  get rowCount(): number { return this._result.rowCount; }
  get fields() { return this._result.fields; }
  get durationMs(): number { return this._result.durationMs; }
}
```

**Update `stream()` to yield `RecordBatch`:**
```typescript
async *stream(
  conn: Connection,
  query: string,
  params?: unknown[],
  options: { batchSize?: number } = {}
): AsyncIterable<import('apache-arrow').RecordBatch> {
  const adapter = this.adapter;
  const batchSize = options.batchSize ?? this.options.batchSize ?? 10_000;

  try {
    if (adapter.streamBatches) {
      // Arrow-native path: yield RecordBatches directly from driver
      yield* adapter.streamBatches(conn, query, params, { batchSize });
    } else {
      // JS fallback: convert row-by-row stream to RecordBatch chunks
      const { batchRows } = await import('./arrow-utils.js');
      const source = adapter.stream(conn, query, params);
      yield* batchRows(source, [], batchSize);
    }
  } catch (err) {
    throw wrapError(err);
  }
}
```


**`streamBatches(conn, query, params?, options?)`:**
```typescript
async *streamBatches(conn, query, params?, options?) {
  const batchSize = options?.batchSize ?? 10_000;
  const prepared = await conn.native.conn.prepare(query);
  // DuckDB's streaming interface: use runAndReadAll chunked, or pending stream API
  const reader = await prepared.runAndReadAll();
  const buf = reader.toArrowBuffer();
  const table = tableFromIPC(buf);
  for (const batch of table.batches) {
    yield batch;
  }
}
```
*Note: if `@duckdb/node-api` exposes a chunk-by-chunk streaming API (e.g., `sendAndConsumeChunks`), use that instead of `runAndReadAll` to avoid full buffering.*

```typescript
async *streamBatches(conn, query, params?, options?) {
  const rs = await conn.native.query({
    query,
    format: 'ArrowStream',
    query_params: paramsToRecord(params),
  });
  // rs.stream() returns AsyncIterable<Buffer[]> of Arrow IPC record batches
  const stream = rs.stream();
  for await (const chunk of stream) {
    // Each chunk may be a Buffer or Uint8Array of Arrow IPC RecordBatch
    const batch = tableFromIPC(Buffer.concat(chunk)).batches[0];
    if (batch && batch.numRows > 0) yield batch;
  }
}
```

### 4.1 Add `--stream` and `--batch-size` to `mdb query`

**File: `apps/cli/src/commands/query.ts`**

Add to yargs builder:
```typescript
.option('stream', {
  type: 'boolean',
  default: false,
  description: 'Stream output with constant memory (one RecordBatch at a time)',
})
.option('batch-size', {
  type: 'number',
  default: 10_000,
  description: 'Rows per Arrow batch in streaming mode',
})
```

In the handler, when `argv.stream`:
```typescript
if (argv.stream) {
  if (argv.format === 'table') {
    process.stderr.write('[maitredb] --stream: table format requires all rows; using ndjson\n');
    argv.format = 'ndjson';
  }
  const batches = executor.stream(conn, sql, [], { batchSize: argv['batch-size'] });
  for await (const batch of batches) {
    const rows = recordBatchToRows(batch);
    const formatter = getFormatter(argv.format as OutputFormat);
    // Emit rows without header repeat (for table/csv) — emit each batch separately
    // For ndjson/raw: natural per-row emission
    process.stdout.write(formatter.format({ rows, fields: [], rowCount: rows.length, durationMs: 0 }));
  }
} else {
  // existing execute() path
}
```
